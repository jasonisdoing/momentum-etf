# 01. 배경과 목표

## 현재 시스템

- 현재 운영 UI는 Python `Streamlit` 기반이다.
- 데이터/도메인 로직은 Python 코드와 MongoDB 중심으로 구성되어 있다.
- 현재 배포 구조는 Python 앱 단일 컨테이너 + `nginx-proxy` 기반이다.

## 문제 인식

사용자는 현재 Streamlit UI에 제약이 많다고 느끼고 있다.

특히 다음 성격의 화면은 Node.js + TypeScript 기반 웹 UI로 옮길 때 장점이 크다고 판단했다.

- 대시보드
- 원금/현금 관리
- 스냅샷
- ETF 마켓

## 중요한 작업 원칙

사용자는 과거에 문서와 계획을 먼저 많이 정리한 뒤,
실제 기술 제약 때문에 프로젝트 방향을 되돌린 경험이 있다.

그래서 이번에는 다음 원칙을 우선한다.

- 상세 계획보다 `최소 기술 검증`을 먼저 한다.
- 실제 가능한 버전을 작게라도 먼저 띄운다.
- 그 결과를 본 뒤 다음 이전 범위를 정한다.

## 이번 작업의 1차 목표

이번 단계의 목표는 기능 완성이 아니다.

이번 단계의 진짜 목표는 아래 한 줄이다.

`같은 도메인에서 Node 앱과 Python 앱이 동시에 안정적으로 뜨는지 검증한다.`

## 이번 단계에서 아직 하지 않을 것

- 전체 메뉴 이전
- Python 로직의 대규모 재작성
- Node에서 실제 비즈니스 기능 완성
- 최종 메뉴/IA의 세부 확장안 확정
