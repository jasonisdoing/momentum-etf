## 명확히 확정된 포인트

- `원금/현금` 기능의 소유권은 Node.js 쪽이다.
- Python에서는 `원금/현금` 메뉴를 제거하는 방향이다.
- 스냅샷은 1차에서 Node.js에 `리스트 중심 단순 화면`으로 먼저 둔다.
- Node 화면은 가능하면 Python 계산 로직 또는 Python API를 재사용한다.
- 아직 이전하지 않은 기능은 Python에서 유지한다.
- 앞으로 Node UI는 `ERP 스타일 기준`으로 구현한다.

## Node UI 스타일 기준

- 랜딩 페이지/마케팅 페이지 느낌보다 업무용 백오피스 UI를 우선한다.
- 카드형 과다 사용보다 테이블, 행 레이아웃, 압축형 폼을 우선한다.
- 특별한 요청이 없으면 기본 표현은 테이블식으로 간다.
- 상단 설명 문구는 최소화한다.
- 저장/오류/상태 피드백은 눈에 잘 띄게 표시한다.
- 상단 메뉴와 본문 작업 영역 구조를 기본으로 한다.
- 루트 `/` 는 아이콘 기반 앱 런처형 홈으로 둔다.
- 정보 밀도와 빠른 입력 효율을 미적 연출보다 우선한다.
