## 명확히 확정된 포인트

- `원금/현금` 기능의 소유권은 Node.js 쪽이다.
- Python에서는 `원금/현금` 메뉴를 제거하는 방향이다.
- 스냅샷은 1차에서 Node.js에 `리스트 중심 단순 화면`으로 먼저 둔다.
- Node 화면은 가능하면 기존 Mongo 스키마와 계산 결과를 재사용한다.
- Python UI는 제거하고, 필요한 Python 코드는 배치/스크립트 용도로만 남긴다.
- 앞으로 Node UI는 `ERP 스타일 기준`으로 구현한다.
- 앞으로 Node UI의 레이아웃/컴포넌트/밀도 기준은 `Tabler 기본 스타일`을 우선 사용한다.
- 커스텀 CSS는 `도메인 의미색`, `양수/음수 값 강조`처럼 꼭 필요한 경우에만 최소한으로 허용한다.
- 화면마다 다르게 보이는 문제를 수동 미세조정으로 맞추지 않고, `전역 공통 규칙을 Tabler 기준으로 표준화`하는 방식으로 해결한다.
- Node 인증은 `Google OAuth + 허용 이메일 화이트리스트` 방식으로 간다.
- Python과 Node를 연결하는 임시 브리지 UI는 우선순위가 아니다.

## Node UI 스타일 기준

- 랜딩 페이지/마케팅 페이지 느낌보다 업무용 백오피스 UI를 우선한다.
- 카드형 과다 사용보다 테이블, 행 레이아웃, 압축형 폼을 우선한다.
- 특별한 요청이 없으면 기본 표현은 테이블식으로 간다.
- 특별한 요청이 없으면 `Tabler 기본 navbar / page-header / card / table / button / form` 조합을 그대로 사용한다.
- 상단 설명 문구는 최소화한다.
- 저장/오류/상태 피드백은 눈에 잘 띄게 표시한다.
- 상단 메뉴와 본문 작업 영역 구조를 기본으로 한다.
- 루트 `/` 는 아이콘 기반 앱 런처형 홈으로 둔다.
- 정보 밀도와 빠른 입력 효율을 미적 연출보다 우선한다.
- 화면별 spacing, 높이, 패딩 차이는 개별 페이지에서 임시로 덧대지 말고 전역 표준 규칙으로 흡수한다.
