# 05. 다음 작업 순서

## 바로 다음 액션

다음 작업은 문서가 아니라 `최소 공존 버전 구현`이다.

## Step 1. 최소 Node 앱 추가

목표:

- TypeScript 기반 Node 앱을 저장소에 추가한다.
- 우선은 화면이 뜨는지만 확인할 최소 라우트만 만든다.

검증 포인트:

- `/dashboard`
- `/cash`
- `/snapshots`
- `/market`

## Step 2. Python 앱을 /py 경로로 띄우는 실행 방식 준비

목표:

- 기존 Python 앱을 유지하되, 하이브리드 실행에서는 `/py` 경로 아래에서 동작하게 만든다.

검증 포인트:

- `/py/rank`
- `/py/system`

## Step 3. nginx 경로 분기 추가

목표:

- `/` 또는 Node 경로는 Node 앱으로 전달
- `/py/*` 는 Python 앱으로 전달

검증 포인트:

- Node와 Python이 동시에 접속 가능
- 라우팅 충돌 없음

## Step 4. 실제 실행 확인

목표:

- 로컬 또는 운영과 유사한 환경에서 실제로 띄워본다.

최소 확인 항목:

- 두 앱이 모두 떠 있는가
- 메인 경로와 `/py/*`가 모두 열리는가
- 정적 리소스 문제는 없는가
- Streamlit 화면이 정상 렌더링되는가

## Step 5. 그 다음에만 할 것

기술 검증이 통과한 뒤에만 아래를 진행한다.

- Python 메뉴 개편 실제 반영
- Node 메뉴에 실제 데이터 연결
- API 범위 상세 설계
- 메뉴 이전 우선순위 세부화
