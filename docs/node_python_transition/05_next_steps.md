# 05. 다음 작업 순서

## 바로 다음 액션

하이브리드 1차 배포와 `벌크 입력` 1차 구현까지 끝났다.
이제 다음 작업은 `남은 Python 메뉴를 Node로 순차 이관`하는 것이다.

## Step 1. 다음 Node 이관 대상 확정

목표:

- 다음으로 옮길 Python 메뉴를 하나 정한다.
- 권장 우선순위는 `종목 관리` -> `삭제된 종목` -> `주별` 순서다.

검증 포인트:

- Streamlit 제약이 큰 화면인가
- 저장 규칙이 명확한가
- 기존 Mongo 스키마를 그대로 재사용할 수 있는가

## Step 2. Node 화면 1차 구현

목표:

- 선택한 Python 메뉴를 Node에서 1차 동작 가능한 화면으로 옮긴다.
- Python과의 임시 연결 UI 대신 Node 단독 화면 완성도를 우선한다.

검증 포인트:

- 실제 데이터 조회 성공
- 저장이 필요한 화면이면 Mongo 반영 성공
- Python 기존 규칙과 결과가 일치하는가

## Step 3. 로컬 개발 검증

목표:

- 로컬에서는 기본적으로 Docker 없이 검증한다.
- Node는 `npm run dev`, Python은 개별 실행으로 빠르게 확인한다.

검증 포인트:

- Node dev 서버에서 `.env`, `zaccounts`, Mongo 연결이 정상인가
- 화면 수정이 즉시 반영되는가

## Step 4. 필요할 때만 하이브리드/배포 회귀 확인

목표:

- 프록시 문제나 배포 구조 영향이 의심될 때만 Docker 하이브리드와 운영 회귀를 다시 본다.

후보:

- `/`
- `/dashboard`
- `/import`
- `/cash`
- `/snapshots`
- `/market`
- `/py/`
