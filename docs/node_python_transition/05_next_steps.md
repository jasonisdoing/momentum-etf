# 05. 다음 작업 순서

## 바로 다음 액션

하이브리드 1차 배포, `Google OAuth 인증`, `종목 관리` 1차 구현까지 끝났다.
이제 다음 작업은 `삭제된 종목`을 Node로 이관하는 것이다.

## Step 1. 삭제된 종목 Node 이관

목표:

- 다음으로 `삭제된 종목`을 Node로 옮긴다.
- 그 다음 우선순위는 `주별` -> `시스템정보/메모` 순서다.

검증 포인트:

- 기존 Mongo 스키마를 그대로 재사용할 수 있는가
- 복구/영구삭제 규칙이 Python과 일치하는가

## Step 2. Node 화면 1차 구현

목표:

- `삭제된 종목`을 Node에서 1차 동작 가능한 화면으로 옮긴다.
- Python과의 임시 연결 UI 대신 Node 단독 화면 완성도를 우선한다.

검증 포인트:

- 실제 데이터 조회 성공
- 복구/삭제 액션이 Mongo 반영에 성공
- Python 기존 규칙과 결과가 일치하는가

## Step 3. 로컬 개발 검증

목표:

- 로컬에서는 기본적으로 Docker 없이 검증한다.
- Node는 `npm run dev`, Python은 개별 실행으로 빠르게 확인한다.

검증 포인트:

- Node dev 서버에서 `.env`, `zaccounts`, Mongo 연결이 정상인가
- 화면 수정이 즉시 반영되는가

## Step 4. 필요할 때만 하이브리드/배포 회귀 확인

목표:

- 프록시 문제나 배포 구조 영향이 의심될 때만 Docker 하이브리드와 운영 회귀를 다시 본다.

후보:

- `/`
- `/dashboard`
- `/import`
- `/cash`
- `/snapshots`
- `/market`
- `/py/`
