# 04. 현재 진행 상태

## 완료된 것

- 기존 메뉴 구조를 분석했다.
- Node와 Python 사이의 메뉴 소유권 초안을 확정했다.
- URL 구조 v1 초안을 확정했다.
- 기술 검증을 계획보다 먼저 하기로 방향을 확정했다.
- 관련 요약 문서를 `docs/`에 추가했다.

## 아직 하지 않은 것

- Node 앱 코드 생성
- 두 앱을 함께 띄우는 nginx 설정 작성
- 하이브리드 docker/실행 설정 추가
- 실제 접속 테스트
- Node/Python/API 경로 충돌 확인

## 현재 상태 한 줄 요약

`아키텍처 방향은 확정되었고, 아직 실제 하이브리드 실행 스파이크는 시작하지 않은 상태다.`

## 지금 시점의 핵심 미해결 기술 이슈

- Streamlit의 base path를 `/py`로 두었을 때 문제가 없는지
- nginx 프록시에서 websocket/정적 경로가 안정적인지
- 기존 단일 Docker 구조를 깨지 않고 하이브리드 실행 세트를 어떻게 둘지

## 중요 참고

사용자 의도는 명확하다.

- 지금은 계획 고도화보다 `실행 가능한 최소 버전`이 먼저다.
- 문서만 더 늘리는 것은 우선순위가 아니다.
- 이 폴더의 문서는 실행 스파이크를 돕기 위한 기준선으로만 사용한다.
