# 전략 로직 상세 (Strategy Logic)

이 문서는 현재 시스템의 의사결정 로직(계좌별 순위 계산)을 설명합니다.

## 1. 공통 실행 원칙

* 순위 화면은 계좌에 등록된 전체 종목 유니버스를 직접 사용합니다.
* 계산 기준은 당일 종가 기준입니다.
* 최신 거래일 판단의 기준 날짜는 모든 시장 공통으로 한국 날짜를 사용합니다.
* 필수 설정이 누락되면 fallback 없이 명시적 에러로 중단됩니다.

## 2. 계좌 순위 계산 로직

계좌(`zaccounts/*`)는 계좌 종목 전체를 유니버스로 사용하고, `MA_TYPE`, `MA_MONTHS` 기준으로 각 종목의 추세 점수를 계산합니다.
버킷은 분류/표시용 참고 정보로만 사용합니다.

### 핵심 파라미터

* `MA_TYPE`: 이동평균 방식 (`SMA`, `EMA`, `WMA`, `DEMA`, `TEMA`, `HMA`, `ALMA`)
* `MA_MONTHS`: 이동평균 기간(개월)

### 동작 요약

1. 계좌 종목 전체의 추세 점수를 계산
2. `MIN_TRADING_DAYS` 이상 데이터가 있는 종목은 가능한 범위에서 점수를 계산한다
3. 점수 계산이 가능한 종목은 점수 내림차순으로 정렬한다
4. 데이터가 부족한 종목은 점수 계산 불가로 처리하고 맨 아래에 둔다
5. 실제 보유 종목은 화면에서 녹색 행으로 강조한다

### 화면 표시 규칙

* 별도 추천 상태는 사용하지 않습니다.
* 실제 보유 여부는 `summary_details` 기반 실보유 데이터로만 판단합니다.
* 결과 테이블은 점수 내림차순입니다.
* `점수` 컬럼은 `현재가`와 `일간(%)` 사이에 위치합니다.

### 검증 규칙

* `MA_TYPE`, `MA_MONTHS`는 계좌 설정에 있어야 함
* 점수 계산 불가 종목은 맨 아래로 보내야 함
* 실제 보유 종목 강조는 순위 계산과 별도로 적용되어야 함
