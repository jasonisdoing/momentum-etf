# 전략 로직 상세 (Strategy Logic)

이 문서는 현재 시스템의 의사결정 로직(계좌별 순위 계산)을 설명합니다.

## 1. 공통 실행 원칙

* 순위 화면은 계좌에 등록된 전체 종목 유니버스를 직접 사용합니다.
* 계산 기준은 당일 종가 기준입니다.
* 최신 거래일 판단의 기준 날짜는 모든 시장 공통으로 한국 날짜를 사용합니다.
* 필수 설정이 누락되면 fallback 없이 명시적 에러로 중단됩니다.
* 순위 비교에 사용하는 `이전순위`는 현재 화면에 적용된 보유보너스점수를 포함한 동일 조건으로 이전 거래일을 다시 계산해 얻습니다.

## 2. 계좌 순위 계산 로직

종목풀(`pools.json`)은 등록된 전체 종목을 유니버스로 사용하고, `MA_TYPE`, `MA_MONTHS` 기준으로 각 종목의 점수를 계산합니다.
버킷은 분류/표시용 참고 정보로만 사용합니다.
`0. 전체` 가상 종목풀은 `all` 설정의 기준과 `all.include`에 명시한 종목풀만 합산합니다.

### 핵심 파라미터

* `MA_TYPE`: 이동평균 타입
* `MA_MONTHS`: 이동평균 기간(개월)
* `all.include`: `0. 전체` 가상 종목풀 포함 대상

### 동작 요약

1. 종목풀 전체 종목에 대해 `MA_TYPE`, `MA_MONTHS` 기준 단일 추세 값을 계산한다
2. 추세 값을 `0` 기준으로 상승/하락 구간으로 나눈다
3. 각 구간 안에서만 상대 순위(백분위)를 계산하고 부호를 유지한다
4. 부호 보존 백분위 점수를 `점수`로 사용한다
5. `MIN_TRADING_DAYS` 이상 데이터가 있는 종목만 점수 계산 대상이 된다
6. `점수` 계산이 가능한 종목은 `점수` 내림차순으로 정렬한다
7. 데이터가 부족한 종목은 `점수` 계산 불가로 처리하고 맨 아래에 둔다
8. 실제 보유 종목은 화면에서 녹색 행으로 강조한다
9. `고정 종목`은 순위 번호 없이 현재 점수 위치만 참고용으로 표시한다

### 화면 표시 규칙

* 실제 보유 여부는 `summary_details` 기반 실보유 데이터로만 판단합니다.
* 결과 테이블은 `점수` 내림차순입니다.
* `고정 종목`은 `📌 고정`으로 표시합니다.
* `현재가` 뒤에 `점수`, `추세` 같은 컬럼이 위치합니다.

### 검증 규칙

* `MA_TYPE`, `MA_MONTHS`는 종목풀 설정에 있어야 함
* `점수` 계산 불가 종목은 맨 아래로 보내야 함
* 실제 보유 종목 강조는 순위 계산과 별도로 적용되어야 함

## 3. 시장지수 추세 (`/market-trend`)

코스피·코스피200·S&P500·나스닥100 지수의 추세를 점수화하고 레짐(국면)을 보여줍니다.
관련 파라미터는 모두 `config.py` 가 단일 진실 소스이며, 프론트에는 API
응답(`/internal/market-trend/defaults`)으로 전달됩니다.

### 추세 점수 (−100 ~ +100)

1. 원본 추세% = `(종가 ÷ MA[MA_TYPE, MA_MONTHS] − 1) × 100` (MA선이면 0, 위면 +, 아래면 −)
2. 최근 12개월 추세% 분포에서 **상위 P 퍼센타일**(`MARKET_TREND_SCORE_ANCHOR_PERCENTILE`, 기본 95)을 +100,
   **하위 (100−P) 퍼센타일**을 −100 으로 두고 정규화 (MA선=0점 고정)
3. 상위 P% 이상은 +100 에 포화. 단발 최대/최소가 아닌 퍼센타일이라 이상치에 둔감하고 자기보정됨
4. **수익률이 아니라 MA 대비 위치의 정규화 점수**임

### 레짐 (상승 / 조정 / 진정 / 하락)

**방향**(MA 위/아래, 추세% 부호) × **가속/감속**의 4단계:

* 가속/감속은 추세%에 최소제곱 직선을 적합한 **회귀 기울기**로 판정합니다 (과거 4주 평균
  비교 방식에서 변경). **비대칭 창** — "상승은 빠르게, 약화는 천천히":
  * 강화: 짧은 창 `MARKET_TREND_REGIME_SLOPE_UP_WINDOW`(기본 7) 기울기 > +DEADBAND → 저점 반등을 빨리 포착
  * 약화: 긴 창 `MARKET_TREND_REGIME_SLOPE_WINDOW`(기본 20) 기울기 < −DEADBAND → 노이즈에 둔감
* `MARKET_TREND_REGIME_SLOPE_DEADBAND`(기본 0.05) 안쪽(데드밴드)이면 **직전 상태를 유지**해
  0 근처 진동의 라벨 휩소를 막습니다.
* 매핑: MA 위+강화=상승, MA 위+약화=조정, MA 아래+강화=진정, MA 아래+약화=하락.
* 레짐 라벨은 추세 점수 계산과 독립입니다 (라벨 안정화 목적).
