# 전략 로직 상세 (Strategy Logic)

이 문서는 현재 시스템의 의사결정 로직(종목풀 랭킹 + 계좌 비중 조절형 리밸런싱)을 설명합니다.

## 1. 공통 실행 원칙

* 추천(`recommend.py`)과 백테스트(`backtest.py`)는 동일 백테스트 엔진(`core/backtest`)을 사용합니다.
* 매매 판단은 당일 종가 기준으로 계산하고, 체결은 다음 거래일 가격 규칙(슬리피지 포함)으로 시뮬레이션합니다.
* 리밸런싱은 `REBALANCE_MODE` 규칙을 따르며, 엔진은 종목 교체를 수행하지 않습니다.
* 필수 설정이 누락되면 fallback 없이 명시적 에러로 중단됩니다.

## 2. 종목풀 랭킹 로직

종목풀(`zpools/*`)은 이동평균 기반 점수로 종목 상대 순위를 계산합니다.

현재 운영 종목풀:
* `kor`: 국내상장 국내 ETF
* `us`: 국내상장 해외 ETF
* `aus`: 호주 ETF

### 핵심 파라미터

* `MA_MONTH`: 이동평균 기간(개월)
* `MA_TYPE`: 이동평균 종류

### 동작 요약

1. 각 종목의 점수를 계산
2. 점수 순으로 정렬
3. `rank_YYYY-MM-DD.log`와 MongoDB `pool_rankings`에 저장

## 3. 계좌 리밸런싱 로직

계좌(`zaccounts/*`)는 계좌 종목 전체를 유니버스로 사용하고, 버킷별 상승 점수 합계에 비례한 동적 `weight`를 목표 비중으로 사용하는 리밸런싱 방식입니다.
백테스트는 "현재 계좌 종목을 처음부터 계속 보유했다면 어땠는가"를 보는 용도로 사용합니다.

### 핵심 파라미터

* `REBALANCE_MODE`: 리밸런싱 주기
* `BUCKET_WEIGHT_MIN`, `BUCKET_WEIGHT_MAX`: 버킷당 최소/최대 비중 가드레일
* `REBALANCE_BUFFER`: 현재 비중과 목표 비중 차이에 대한 매매 유보 버퍼

### 동작 요약

1. 계좌 종목 전체의 모멘텀 점수를 계산
2. 버킷별로 상승 점수만 합산해 버킷 점수를 만든다
3. 버킷 점수에 비례해 버킷 목표 비중을 산출하고, `BUCKET_WEIGHT_MIN/MAX` 가드레일을 적용한다
4. 각 버킷 비중은 해당 버킷 종목 수 `N`으로 균등 분배한다
5. 리밸런싱 시점마다 동일 규칙으로 목표 비중을 다시 계산하고 재조정

### 상태 및 문구 규칙

* 상태는 `BUY`, `HOLD`만 사용합니다.
* `BUY`는 백테스트 시작 첫 편입일 또는 상장 후 첫 편입일에만 표시합니다.
* 비중 확대/축소는 상태를 바꾸지 않고 `HOLD`를 유지하며, 문구에만 예정/실행 비중조절을 표시합니다.

### 검증 규칙

* 가드레일 조합은 계좌 종목 수와 양립 가능해야 함
* 산출된 `weight` 합계는 1.0이어야 함
* 수동 종목 비중 입력은 사용하지 않음

## 4. 튜닝 로직

* 계좌 튜닝: `REBALANCE_MODE` 중심으로 탐색
* 종목풀 랭킹 파라미터(`months`, `ma_type`)는 `zpools/*/config.json`에서 관리
* 최적 결과는 계좌 `config.json`의 `strategy`에 반영됩니다.
  * 예: `strategy.REBALANCE_MODE`, `strategy.TUNE_MONTHS`
* 별도의 보유 종목 수 한도 파라미터는 사용하지 않습니다. 계좌 유니버스 전체가 항상 계산 대상입니다.
