# 사용자 가이드 (User Guide)

이 문서는 Momentum ETF 시스템의 설치, 설정, 실행 방법을 설명합니다.

현재 이 시스템은 한국 상장 ETF와 호주 ETF를 순위 분석 대상으로 사용합니다.
계좌 설정의 `country_code`는 `kor` 또는 `au`만 허용합니다.

## 1. 설치 및 실행

### 필수 요구사항
*   Python 3.9 이상
*   Docker (선택 사항, 배포 시 필요)

### 실행 명령어

**웹 실행**
```bash
cd web
npm run dev
```

현재 운영 계좌:
* `kor_account`, `isa_account`, `pension_account`, `core_account`, `aus_account`

## 2. 설정 가이드

### 기본 설정 (`config.py`)
시스템 전반에 걸친 기본 설정을 변경할 수 있습니다.
*   `SLACK_BOT_TOKEN`: 알림을 봇을 통해 발송하기 위한 슬랙 봇 토큰
*   `SLACK_CHANNEL_ID`: 알림을 받을 슬랙 채널 ID

### 계좌 설정 (`zaccounts/<order>_<account_id>/config.json`)
계좌는 등록된 전체 종목을 유니버스로 사용합니다. `MA_TYPE`, `MA_MONTHS` 기준으로 추세 점수를 계산하고, 고점점수를 반영한 통합점수 내림차순으로 순위를 보여줍니다. 버킷은 분류와 표시용 참고 정보입니다.
계좌 `country_code`는 현재 `kor` 또는 `au`만 사용합니다.

```json
{
  "icon": "🇰🇷",
  "name": "국내 계좌",
  "country_code": "kor",
  "currency": "KRW",
  "MA_TYPE": "ALMA",
  "MA_MONTHS": 6
}
```

계좌별 종목은 MongoDB `stock_meta` 컬렉션에서 관리하며, 웹 UI의 **종목 관리** 화면에서 추가/삭제/편집합니다.

## 3. 대시보드 및 계좌 관리

### 대시보드 구성
계좌 화면은 다음 세 가지 보기로 구성됩니다.
*   **1. 순위**: 개별 종목의 현재가, 통합점수, 추세점수, 고점점수, 일간 수익률, RSI, 추세 차트를 확인합니다.
    상단에서 기준일을 바꾸면 해당 날짜까지의 데이터로 순위를 다시 계산합니다. 오늘을 선택한 경우에는 거래일 여부와 관계없이 최신 실시간 스냅샷(휴일에는 직전 거래일 기준 마지막 값)을 반영하며, 토글로 **누적** 수익률 보기와 **월별** 수익률 보기(최근 13개월)를 전환할 수 있습니다.
    순위 표는 AG Grid 기반으로 전환되어 카드 내부 스크롤과 헤더 클릭 정렬을 사용합니다.
*   **2. 종목 관리**: 계좌 활성 종목을 직접 관리합니다. `등록일`은 해당 종목이 현재 활성 상태로 들어온 날짜이며, 삭제 후 복구하면 복구한 날짜로 다시 표시됩니다. 표는 AG Grid 기반으로 전환되어 카드 내부 스크롤과 헤더 클릭 정렬을 사용합니다. 활성 탭은 그리드 바로 위 `추가 / 저장 / 삭제` 버튼을 사용하며, 새 종목은 임시 행에서 티커 확인 후 저장하고 기존 종목은 버킷을 그리드에서 직접 바꾼 뒤 저장합니다. 삭제 탭은 체크박스로 복구/영구 삭제 대상을 선택합니다.
*   **3. 삭제된 종목**: **종목 관리** 화면 안의 `삭제된 종목` 탭에서 삭제된 종목을 확인하고 복구합니다. 체크박스로 여러 종목을 선택한 뒤 상단 `선택 복구 / 영구 삭제` 버튼으로 처리합니다.
*   **4. 메모**: 계좌별 고정 메모를 관리합니다.
*   **5. 자산 관리**: 자산 화면은 부모/자식 구조입니다. 부모 테이블에서 5개 계좌의 `원금`, `현금`을 직접 수정하고, 호주 계좌는 `Intl Value`, `Intl Change`도 함께 수정할 수 있습니다. 부모 변경사항은 상단 오른쪽 녹색 `저장` 버튼으로 반영됩니다.
    계좌 row를 클릭하면 하위에 해당 계좌의 종목 테이블이 펼쳐지며, 자식 테이블에서는 `수량`, `매입 단가`, `목표비중`을 직접 수정하고 `추가 / 저장 / 삭제`를 별도로 사용합니다. 목표 관련 값(`목표수량`, `예상변경수량`, `목표금액`)은 편집 중 즉시 미리보기로 반영됩니다.
    데스크탑에서는 부모/자식 그리드 모두 카드 내부 영역에서 스크롤됩니다.
*   **6. ETF 마켓**: AG Grid 기반 조회형 화면으로 전환되어, 기본 제외 필터를 유지하면서도 필터 결과 전체 행을 한 번에 표시합니다. 데스크탑에서는 카드 내부 그리드만 스크롤하고, 헤더 클릭 정렬과 KIS 마스터 기반 전체 탐색이 가능합니다.
*   **7. 주별**: 주별 집계 화면도 AG Grid 기반으로 전환되어 카드 내부 스크롤과 헤더 클릭 정렬을 사용합니다. 메인 헤더 왼쪽에는 `핵심만 보기 / 전체 보기 / 이번주 데이터 집계`, 오른쪽에는 `활성 주차 / 행` 정보가 표시되며, 수정 가능한 셀은 그리드에서 직접 편집하고 그 아래 오른쪽 `저장` 버튼으로 반영합니다.
*   데스크탑 화면 상단에는 사이드바 접기/펼치기 버튼이 있으며, 더 넓게 보고 싶을 때 좌측 메뉴를 숨길 수 있습니다.
*   실제 보유 종목은 녹색 행으로 표시됩니다.

계좌 메모는 MongoDB `account_notes` 컬렉션에 계좌별 최신본 1개만 저장합니다.

### 계좌 관리 (원금 및 현금)
사이드바의 **[계좌 관리]** 메뉴를 통해 각 계좌의 기초 데이터를 수정할 수 있습니다.
*   **💵 원금 및 현금 관리**: 각 계좌의 '총 투자 원금'과 '남은 현금 잔고'를 입력합니다.
*   입력된 데이터는 실시간으로 대시보드의 **총자산 및 수익률 계산**에 반영됩니다.
*   일별 스냅샷(`daily_snapshots`)의 금액 필드는 모두 KRW 기준 정수로 저장하고 표시합니다.
*   스냅샷 화면에서는 날짜 row를 클릭하면 같은 row 아래에 계좌별 상세가 펼쳐집니다.

### 캐시 알림 (Alerts)
순위 화면에서 가격 캐시가 누락된 종목이 있으면 토스트 경고가 표시됩니다.
*   누락된 종목명과 티커가 구체적으로 안내됩니다.
*   **종목 관리** 화면에서 해당 종목의 편집 모달을 열고 **메타/캐시 새로고침** 버튼을 클릭하여 갱신합니다.
*   주별 집계 시에도 캐시 누락 종목이 있으면 잘못된 데이터로 덮어쓰지 않도록 집계가 중단됩니다.

## 4. 결과 해석

### 로그 파일
실행 중 발생하는 상세 로그는 `logs/` 폴더에 저장됩니다. 오류 발생 시 이 파일을 확인하세요.
