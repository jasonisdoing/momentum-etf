# 사용자 가이드 (User Guide)

이 문서는 Momentum ETF 시스템의 설치, 설정, 실행 방법을 설명합니다.

현재 이 시스템은 한국 상장 ETF와 호주 ETF를 순위 분석 대상으로 사용합니다.
계좌 설정의 `country_code`는 `kor` 또는 `au`만 허용합니다.

## 1. 설치 및 실행

### 필수 요구사항
*   Python 3.9 이상
*   Docker (선택 사항, 배포 시 필요)

### 실행 명령어

**웹 실행**
```bash
cd web
npm run dev
```

현재 운영 계좌:
* `kor_account`, `isa_account`, `pension_account`, `core_account`, `aus_account`

## 2. 설정 가이드

### 기본 설정 (`config.py`)
시스템 전반에 걸친 기본 설정을 변경할 수 있습니다.
*   `SLACK_BOT_TOKEN`: 알림을 봇을 통해 발송하기 위한 슬랙 봇 토큰
*   `SLACK_CHANNEL_ID`: 알림을 받을 슬랙 채널 ID

### 종목풀 설정 (`ztickers/<order>_<ticker_type>/config.json`)
종목풀은 등록된 전체 종목을 유니버스로 사용합니다. `MA_RULES`의 각 규칙별 추세를 계산한 뒤, 각 규칙을 `0` 기준 상승/하락으로 나누고 구간 내 상대 순위(백분위)를 부호를 유지한 채 합산한 `점수` 내림차순으로 순위를 보여줍니다. 버킷은 분류와 표시용 참고 정보입니다.
계좌 `country_code`는 현재 `kor` 또는 `au`만 사용합니다.

```json
{
  "icon": "🇰🇷",
  "name": "국내상장 국내",
  "country_code": "kor",
  "currency": "KRW",
  "MA_RULES": [
    { "order": 1, "MA_TYPE": "SMA", "MA_MONTHS": 10 },
    { "order": 2, "MA_TYPE": "ALMA", "MA_MONTHS": 3 }
  ],
  "RANK_RECOMMEND_SIMILARITY_LOOKBACK_DAYS": 60,
  "RANK_RECOMMEND_SIMILARITY_THRESHOLD": 0.95,
}
```

계좌별 종목은 MongoDB `stock_meta` 컬렉션에서 관리하며, 웹 UI의 **종목 관리** 화면(`/stocks`) `관리모드`에서 추가/삭제/편집합니다.

## 3. 대시보드 및 계좌 관리

### 대시보드 구성
계좌 화면은 다음 세 가지 보기로 구성됩니다.
*   **홈**: 전체 자산 핵심 지표, 구성 비중 차트, 계좌별 요약 테이블을 보여주는 기본 진입 화면입니다.
*   **보유종목**: 실제 보유 종목을 전 계좌 기준으로 합친 뒤 `상승 / 하락` 두 열 카드로 보여주고, 각 열 안에서 버킷별 섹션으로 다시 나눕니다. 같은 티커를 여러 계좌에서 보유한 경우 현재가는 하나만 표시하며 평가금/손익/매입금은 티커 기준으로 합산합니다. 각 버킷 안에서는 평가금이 큰 종목부터 위에 배치합니다.
*   **대시보드** 하단의 **계좌별 요약** 테이블에서는 계좌 설정의 `URL`이 있는 계좌명만 새 창 링크로 열립니다. `URL`이 없는 계좌명은 일반 텍스트로 표시됩니다.
*   **1. 종목 관리**: 개별 종목의 현재가, 점수, `추세1`, `추세2`, 일간 수익률, RSI, 추세 차트를 확인합니다.
    메뉴명 오른쪽에는 `점수 양수 / 기준 / 총 개수` 요약이 표시되고, 카드 메인 헤더에는 실제 제어 옵션만 남깁니다.
    메인 헤더의 기준일, 종목 타입, 추세1/추세2는 `라벨 위 / 입력 아래` 구조로 표시됩니다.
    상단에서 기준일을 바꾸면 해당 날짜까지의 데이터로 순위를 다시 계산합니다. 오늘을 선택한 경우에는 거래일 여부와 관계없이 최신 실시간 스냅샷(휴일에는 직전 거래일 기준 마지막 값)을 반영하며, `수익률 보기` 토글로 **누적수익률**, **월별수익률**(최근 13개월), **정보**를 전환할 수 있습니다. `정보` 보기에는 **배당률**, **보수**, **순자산총액**, **상장일** 컬럼이 표시되며, 값은 종목 메타 캐시(`stock_cache_meta.meta_cache`)를 기준으로 채웁니다.
    `중복제거` 스위치는 기본적으로 켜져 있으며, `중복` 컬럼 값이 있는 종목을 화면에서 숨깁니다.
    `추세1`, `추세2`는 각각 이동평균 타입과 개월 수를 독립적으로 변경할 수 있습니다.
    표의 `순위`는 현재 알고리즘 순위이며, `이전순위`는 같은 조건에서 이전 거래일 알고리즘 순위를 비교해 `-`, `▲`, `▼`로 변화를 표시합니다. 사용자가 다른 컬럼으로 정렬해도 이 두 값은 원래 알고리즘 순위를 유지합니다.
    테스트용 `중복`, `중복상세` 컬럼은 최근 N거래일 일간 수익률 상관계수 기준 유사 그룹을 만들고, 대표 종목 대비 제외된 종목에 `1위 중복` 같은 요약과 `대표종목명(티커) 중복` 상세를 함께 표시합니다.
    순위 표는 AG Grid 기반으로 전환되어 카드 내부 스크롤과 헤더 클릭 정렬을 사용합니다.
    상단의 `화면 모드` 토글로 `순위모드 / 관리모드`를 전환할 수 있습니다. `순위모드`에서는 `추세1`, `추세2`, `수익률 보기`, `중복제거`가 보이고, `관리모드`에서는 체크박스 컬럼과 `추가 / 저장 / 삭제` 버튼이 나타납니다. 표의 **티커** 또는 **종목명**을 클릭하면 개별종목 페이지(`/ticker?ticker=...`)로 이동하여 해당 종목이 바로 조회됩니다. `관리모드`에서 `추가`를 누르면 임시 행이 생기고, 여기서 티커 확인 후 저장할 수 있습니다. 기존 종목은 `버킷` 컬럼을 직접 수정한 뒤 저장합니다. 선택 후 `삭제`를 누르면 즉시 하드 딜리트되며, 삭제 사유나 삭제일자를 따로 남기지 않습니다.
*   **1-1. 개별종목**: 화면 상단 헤더의 `티커/종목명` 버튼을 누르면 검색 패널이 아래로 펼쳐지며, 그 안에서 티커 또는 종목명을 검색할 수 있습니다. 종목 관리, 자산 관리 같은 화면에서 티커나 종목명을 클릭해도 개별종목으로 이동합니다. 검색 패널에는 자동완성 결과, 최근 검색, 그리고 3개 티커 타입별 `일간(%)` 상위 5개 종목을 나눠 보여주는 **급상승** 목록이 함께 표시됩니다. 급상승 제목에는 가격 캐시 기준 시각이 `급상승(2026-04-09(목) 오전 9:10 기준)` 형태로 함께 표시됩니다. 어느 메뉴에서든 원하는 종목을 선택하면 개별종목 화면으로 이동하며, `/ticker?ticker=...`처럼 직접 진입한 종목도 성공적으로 로드되면 최근 검색에 자동 추가됩니다. 선택한 종목명, 티커, 현재가, 등락률은 메뉴명 오른쪽이 아니라 본문 상단의 큰 헤더로 표시됩니다. 캔들스틱 차트는 `일 / 주 / 월` 봉 기준으로 전환할 수 있고, 거래량과 이동평균선 5/20/60/120도 함께 표시합니다. 차트는 현재 보이는 범위를 기준으로 고점 대비 등락률과 저점 대비 등락률도 차트 안 텍스트로 함께 보여줍니다. 한국 ETF는 상단 영역에서 차트 왼쪽, 우측에는 종목 캐시에 저장된 구성종목을 함께 표시합니다. 우측 패널 제목은 비중 값이 하나라도 있는 ETF면 `구성종목비중`, 비중 값이 모두 없으면 `구성종목`으로 바뀝니다. 패널은 `종목코드 / 종목명 / 현재가 / 일간(%)`를 보여주고, 비중 값이 있는 ETF만 `비중` 컬럼을 추가로 표시합니다. 네이버 응답에 포함된 `원화현금`, `설정현금액` 같은 현금성 항목도 그대로 표시하며, 이런 항목은 가격과 일간(%) 없이 비중만 보여줄 수 있습니다. 국내 구성종목만 있으면 `적용일 YYYY-MM-DD`, 국내/해외가 섞이면 `적용일 YYYY-MM-DD(해외 가격 기준 YYYY-MM-DD)`, 해외 구성종목만 있으면 `해외 가격 기준 YYYY-MM-DD`를 메타 문구로 표시합니다. 해외 구성종목은 캐시에 저장된 Reuters 코드와 심볼을 바탕으로 `AMAT` 같은 표시용 종목코드를 사용하고, 원본 ISIN은 내부 필드 `raw_code`에 보존합니다. 해외 구성종목의 `현재가 / 일간(%)`는 응답 시점에 Yahoo 가격을 조회하며, 서비스 메모리 TTL 캐시를 적용합니다. 차트는 `일` 선택 시 최근 1년, `주` 선택 시 최근 2년, `월` 선택 시 전체 범위를 기본으로 보여주며, 이후 확대/축소로 범위를 조정할 수 있습니다. 하단에는 일별 시세 테이블(날짜, 종가, 등락률, 거래량)과 월별 집계 테이블(년월, 월말 종가, 월간 등락률, 월간 거래량)을 함께 표시합니다. 차트와 테이블은 `CACHE_START_DATE`부터 최신 거래일까지의 캐시 범위를 기준으로 보여줍니다. 데스크톱에서는 일별/월별이 좌우 2열로 보이고, 모바일에서는 일별 아래에 월별이 이어서 표시됩니다.
*   **4. 메모**: 계좌별 고정 메모를 관리합니다. 메뉴명 오른쪽에는 현재 선택 계좌와 저장 상태가 표시됩니다. 메인 헤더의 `계좌` 선택은 `라벨 위 / 입력 아래` 구조로 표시되고, `메모 저장` 버튼은 보조 액션 헤더 오른쪽에 배치됩니다.
*   **5. 자산 관리**: 자산 화면은 부모/자식 구조입니다. 상단 메뉴명 오른쪽에는 `총 자산 / 평가액 / 현금 / 계좌수` 요약이 표시되고, 부모 카드 안에서는 중복 제목 없이 바로 부모 테이블이 나옵니다. 부모 테이블에서 5개 계좌의 `원금`, `현금`을 직접 수정하고, 호주 계좌는 `Intl Value`, `Intl Change`도 함께 수정할 수 있습니다. 부모 셀 수정은 별도 저장 버튼 없이 자동 저장되며, 실패한 경우에만 토스트로 알립니다.
    계좌 row를 클릭하면 하위에 해당 계좌의 종목 테이블이 펼쳐지며, 자식 테이블에서는 `수량`, `매입 단가`, `목표비중`을 직접 수정합니다. 기존 종목 수정은 자동 저장되고, 자동 저장이 성공하면 어떤 값이 바뀌었는지 짧은 토스트로 안내합니다. `저장` 버튼은 대기 중인 수정/순서 변경/추가를 즉시 반영할 때 사용할 수 있습니다. `추가`와 `삭제`는 계속 명시 액션으로 처리합니다. 자식의 `목표비중`을 바꾸면 부모의 `목표비중합`도 바로 같이 갱신됩니다. 자식 테이블의 `종목코드`, `종목명`을 클릭하면 개별종목 페이지(`/ticker?ticker=...`)로 이동할 수 있습니다. 자식 테이블 `비중`은 한국 계좌는 `총자산`, 호주 계좌는 `총자산 - IS 평가금`을 분모로 계산하며, 호주 `IS` 행 자체는 `비중`에서 제외합니다.
    자식 테이블은 드래그로 행 순서를 바꿀 수 있으며, 변경한 순서는 조용히 저장되어 다음에도 유지됩니다.
    데스크탑에서는 부모/자식 그리드 모두 카드 내부 영역에서 스크롤됩니다.
*   **6. ETF 마켓**: AG Grid 기반 조회형 화면으로 전환되어, 기본 제외 필터를 유지하면서도 필터 결과 전체 행을 한 번에 표시합니다. 상단 메인 헤더에는 `티커/종목명`, `시가총액(억)`, `거래량(주)` 필터가 라벨과 placeholder와 함께 표시됩니다. 메뉴명 오른쪽에는 `총 / 전체 / KIS 마스터 갱신 시각`이 표시됩니다. 데스크탑에서는 카드 내부 그리드만 스크롤하고, 헤더 클릭 정렬과 KIS 마스터 기반 전체 탐색이 가능합니다.
*   **7. 주별**: 주별 집계 화면도 AG Grid 기반으로 전환되어 카드 내부 스크롤과 헤더 클릭 정렬을 사용합니다. 메뉴명 오른쪽에는 `활성 주차 / 행 / 변경` 요약이 표시되고, 메인 헤더의 `보기 방식`, `집계`는 `라벨 위 / 입력 아래` 구조로 표시됩니다. 수정 가능한 셀은 그리드에서 직접 편집하고 그 아래 오른쪽 `저장` 버튼으로 반영합니다.
*   데스크탑 화면 상단에는 사이드바 접기/펼치기 버튼이 있으며, 더 넓게 보고 싶을 때 좌측 메뉴를 숨길 수 있습니다.
*   실제 보유 종목은 녹색 행으로 표시됩니다.

계좌 메모는 MongoDB `account_notes` 컬렉션에 계좌별 최신본 1개만 저장합니다.

### 계좌 관리 (원금 및 현금)
사이드바의 **[계좌 관리]** 메뉴를 통해 각 계좌의 기초 데이터를 수정할 수 있습니다.
*   **💵 원금 및 현금 관리**: 각 계좌의 '총 투자 원금'과 '남은 현금 잔고'를 입력합니다.
*   입력된 데이터는 실시간으로 대시보드의 **총자산 및 수익률 계산**에 반영됩니다.
*   일별 스냅샷(`daily_snapshots`)의 금액 필드는 모두 KRW 기준 정수로 저장하고 표시합니다. 외화 계좌 현금은 스냅샷 생성 시 해당 환율로 KRW 환산 후 반영하며, 계좌별 스냅샷에는 `cash_balance_native`, `cash_currency`를 함께 저장해 원본 외화 현금도 백업합니다.
*   스냅샷 화면에서는 날짜 row를 클릭하면 같은 row 아래에 계좌별 상세가 펼쳐집니다.
*   스냅샷 화면 메뉴명 오른쪽에는 최신 스냅샷 날짜와 총 날짜 수가 표시됩니다.

### 캐시 알림 (Alerts)
순위 화면에서 가격 캐시가 누락된 종목이 있으면 토스트 경고가 표시됩니다.
*   누락된 종목명과 티커가 구체적으로 안내됩니다.
*   필요하면 로컬에서 `python scripts/stock_meta_cache_updater.py` 또는 `python scripts/stock_price_cache_updater.py`를 실행해 정리합니다.
*   주별 집계 시에도 캐시 누락 종목이 있으면 잘못된 데이터로 덮어쓰지 않도록 집계가 중단됩니다.

### 종목 캐시 개념
이 프로젝트에서 **종목 캐시**는 두 가지를 함께 가리킵니다.

*   **가격 캐시**: OHLCV, 실시간 현재가 같은 자주 변하는 값
*   **메타 캐시**: 상장일, 배당률, 보수, 순자산총액, ETF 구성종목 같은 잘 안 바뀌는 값

사용자 화면에서는 이 둘을 묶어 `종목 캐시`로 이해하면 됩니다.

원칙은 **실시간 가격데이터를 제외한 모든 값은 종목 캐시에 넣는다**는 것입니다. 화면은 메타/구성종목을 종목 캐시에서 한꺼번에 읽어 속도를 유지하고, 실시간성이 필요한 현재가/등락률만 별도로 조회합니다.

### 종목 메타 캐시 갱신
저빈도 ETF 메타 정보는 별도 메타 캐시로 저장합니다.

```bash
./.venv/bin/python scripts/stock_meta_cache_updater.py
./.venv/bin/python scripts/stock_meta_cache_updater.py kor_us
```

이 스크립트는 종목타입별 활성 종목을 순회하면서 한국 ETF의 네이버 `ETFBase`, `ETFDividend`, `ETFComponent`를 조회해 다음 값을 `stock_cache_meta`에 저장합니다.

`meta_cache`
* `배당률`
* `최근 12개월 분배금`
* `최근 배당락일`
* `보수`
* `순자산총액`
* `상장일`
* `운용사`
* `기초지수`

`holdings_cache`
* `구성종목 목록`
* `비중`
* `구성종목 수량`
* `구성종목 평가금액`
* `원본 ISIN / Reuters 코드 / 표시용 심볼`
* `구성종목 기준일`

## 4. 결과 해석

### 로그 파일
실행 중 발생하는 상세 로그는 `logs/` 폴더에 저장됩니다. 오류 발생 시 이 파일을 확인하세요.

### 한국 ETF 구성종목 조회
한국 ETF 구성종목 비중은 `stock_cache_meta.holdings_cache`를 우선 사용합니다.

* `scripts/stock_meta_cache_updater.py`가 네이버 `ETFComponent`를 조회해 구성종목 캐시에 저장합니다.
* `/ticker`는 저장된 구성종목 캐시를 읽고, 현재가와 일간(%) 같은 실시간 가격만 별도로 조회합니다.
* 해외 구성종목 가격은 Yahoo를 이용해 응답 시점에 조회하며, 서비스 메모리 TTL 캐시를 사용합니다.
