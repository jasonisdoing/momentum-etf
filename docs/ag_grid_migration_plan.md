# AG Grid 전환 진행상황

## 1. 설명

이 문서는 MUI Community `DataGrid` 기반 화면을 AG Grid로 전환하는 현재 진행상황만 간단히 정리한 문서입니다.

목표:

1. 100행 제한 없이 전체 행 표시가 필요한 화면을 안정적으로 처리한다.
2. 헤더 클릭 정렬, 내부 스크롤, 선택, 편집 같은 그리드 기능을 화면 성격에 맞게 유지한다.
3. 조회형과 CRUD형 화면을 각각 재사용 가능한 패턴으로 정리한다.

## 3. 화면별 특징

### `/market`

- 조회형 대표 화면
- AG Grid Theming API 적용 완료
- 카드 내부 그리드 스크롤 사용
- 헤더 클릭 정렬 가능
- 필터 결과 전체 행 표시 가능
- 조회형 공통 패턴의 기준 화면으로 사용

### `/assets`

- CRUD형 대표 화면
- AG Grid Theming API 적용 완료
- 카드 내부 그리드 스크롤 사용
- `수량`, `매입 단가`, `목표비중` 셀 더블클릭 편집 가능
- 수정 내용은 상단 `저장` 버튼으로 일괄 반영
- 상단 `추가`, `저장`, `삭제` 버튼 사용
- 삭제는 체크박스 선택 후 일괄 처리
- 목표수량, 예상변경수량, 목표금액은 편집 중 즉시 미리보기 반영
- 수정 가능한 셀과 실제 수정된 셀을 색상으로 구분

### `/rank`

- 조회형
- 동적 컬럼이 많음
- `누적 / 월별`, 기준일, 검색, 행 강조가 함께 있음
- 조회형 전환 시 난이도 높은 화면

### `/stocks`

- CRUD형
- 활성/삭제 탭 구조
- 선택, 복구, 영구 삭제, 추가 흐름 존재

### `/deleted`

- CRUD형
- 선택 후 복구/삭제 중심 화면

### `/weekly`

- CRUD형
- 인라인 편집보다 모달 연계 비중이 높음

### `/snapshots`

- 조회형
- 상단/하단 2개 그리드가 연동됨

### `/cash`

- CRUD형
- 구조는 단순하지만 저장 흐름이 있음

### `/system`

- 조회형
- 단순 요약성 표 위주
