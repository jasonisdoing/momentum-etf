description: DB 마이그레이션 시 반드시 따라야 할 순서 — 비교 확인 → 백업 → 실행
type: feedback
---

DB 마이그레이션은 반드시 다음 순서를 따를 것:
1. 변경 전/후 차이를 먼저 조회하여 사용자에게 보여주고 확인받기
2. 백업 컬렉션 생성
3. 마이그레이션 실행
4. 며칠 후 백업 제거

**Why:** 사용자가 오랜 개발 경험으로 확립한 절차. 백업 없이 바로 실행하면 데이터 유실 시 복구 불가.
**How to apply:** DB를 변경하는 스크립트는 절대 바로 실행하지 않는다. 항상 dry-run/비교부터.
